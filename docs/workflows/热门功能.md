# 热门功能

> 来源：[shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) — HOT 部分

Claude Code 最新和最热门的功能追踪。

---

## /btw — 侧链对话

- 在 Claude 工作时发起不中断主上下文的侧链提问
- 不会污染主对话历史
- 适合快速查询、确认信息

```
/btw 这个函数的返回类型是什么？
```

---

## Code Review — 多代理 PR 分析

- 通过 GitHub App 自动分析 Pull Request
- 使用多个代理并行审查不同方面：
  - Bug 检测
  - 安全漏洞扫描
  - 回归检测
  - 代码风格检查

### 替代产品
- Greptile
- CodeRabbit

---

## Scheduled Tasks — 定时任务

- 使用 `/loop` 按计划运行提示
- 最长运行 3 天
- 可轮询部署状态、构建结果

```
/loop 每 5 分钟检查一次部署状态
```

---

## Voice Mode — 语音模式

- 通过 `/voice` 启用
- 语音输入提示词
- 适合快速口述需求

### 替代产品
- Wispr Flow

---

## Simplify & Batch — 简化与批量

### `/simplify`
- 审查变更代码的复用性、质量和效率
- 自动重构消除重复代码

### `/batch`
- 跨多个文件批量运行命令
- 适合大规模重命名、格式化等操作

---

## Agent Teams — 多代理并行

- 多个 Claude 代理在同一代码库上并行工作
- 通过环境变量协调
- 每个代理可使用独立 git worktree

### 关键环境变量
| 变量 | 描述 |
|------|------|
| `CLAUDE_AGENT_TEAM_ID` | 团队标识 |
| `CLAUDE_AGENT_TEAM_MEMBER_ID` | 成员标识 |

---

## Remote Control — 远程控制

- 通过 `/remote-control` 或 `/rc` 启用
- 从任何设备（手机、平板、另一台电脑）继续本地会话
- 支持移动伴侣应用 `/mobile`

---

## Git Worktrees — 并行开发

- Claude Code 内置 git worktree 支持
- 每个任务在独立 worktree 中工作
- 避免分支切换的上下文丢失
- 通过 `--worktree` 标志或子代理 `isolation: worktree` 启用

### 使用方式
```bash
# CLI 启动
claude --worktree

# 子代理定义
---
isolation: worktree
---
```

---

## Ralph Wiggum Loop — 自主开发循环

- 社区贡献的插件
- 设定目标后 Claude 自主迭代直到完成
- 适合明确定义的任务
- 自动运行测试验证结果

### 工作原理
1. 定义清晰目标和验收标准
2. Claude 开始自主循环
3. 每次迭代：编码 → 测试 → 评估
4. 通过测试则完成，否则继续迭代
