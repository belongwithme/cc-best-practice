# 开发工作流

> 来源：[shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) — DEVELOPMENT WORKFLOWS 部分

收集了多种 Claude Code 实战工作流方案。

---

## 1. Cross-Model 工作流

**Claude Code + OpenAI Codex 协作**

使用多个 AI 模型协作完成任务：
- Claude Code 负责写代码
- OpenAI Codex 负责审查和优化
- 两个模型互相检查，减少单一模型的盲点

### 适用场景
- 复杂的重构任务
- 需要多角度审查的关键代码
- 高安全要求的功能实现

## 2. RPI 工作流（Research → Plan → Implement）

三阶段隔离工作流：

### 阶段 1：Research（研究）
```
claude --permission-mode plan
> 研究项目结构、现有代码模式和依赖关系
> /compact
```

### 阶段 2：Plan（规划）
```
> /plan 设计实现方案
> 生成详细的实施步骤和文件列表
> /compact
```

### 阶段 3：Implement（实施）
```
> 按计划逐步实现
> 每完成一步做 git commit
> 最后运行测试验证
```

### 关键原则
- 每个阶段结束后 `/compact` 压缩上下文
- 研究阶段使用只读模式防止误修改
- 规划阶段产出具体的文件和代码变更列表
- 实施阶段逐步提交，方便回退

## 3. Ralph Wiggum Loop

**长时间自主开发循环**

| 步骤 | 操作 |
|------|------|
| 1 | 设定清晰目标和验收标准 |
| 2 | 启动 Claude Code 自主循环 |
| 3 | Claude 持续迭代直到满足目标 |
| 4 | 自动运行测试验证 |
| 5 | 如未通过，继续迭代 |

### 适用场景
- 明确定义的功能实现
- 有完善测试覆盖的项目
- 需要长时间迭代的复杂任务

## 4. Agent Teams 工作流

**多代理并行开发**

```
# 终端 1：前端代理
CLAUDE_AGENT_TEAM_ID=feature-xyz claude --worktree

# 终端 2：后端代理
CLAUDE_AGENT_TEAM_ID=feature-xyz claude --worktree

# 终端 3：测试代理
CLAUDE_AGENT_TEAM_ID=feature-xyz claude --worktree
```

### 关键配置
- 使用 `CLAUDE_AGENT_TEAM_ID` 协调多个代理
- 每个代理在独立 git worktree 中工作
- 通过 tmux 管理多个终端会话
- 代理间通过 git 共享代码变更

## 选择建议

| 场景 | 推荐工作流 |
|------|-----------|
| 简单功能实现 | 直接使用 Claude Code |
| 需要多角度审查 | Cross-Model 工作流 |
| 复杂功能/不熟悉的代码库 | RPI 工作流 |
| 明确定义、有测试的任务 | Ralph Wiggum Loop |
| 大型功能/多模块协作 | Agent Teams 工作流 |
