# 编排工作流：Command → Agent → Skill

> 来源：[orchestration-workflow](https://github.com/shanraisshan/claude-code-best-practice/tree/main/orchestration-workflow)

本文档描述了 Claude Code 中的 **Command → Agent (with skill) → Skill** 编排模式，通过一个天气数据获取和 SVG 渲染系统来演示。

---

## 系统概述

该天气系统展示了两种不同的 Skill 模式：

- **Agent Skills（预加载技能）**：`weather-fetcher` 在启动时注入到 `weather-agent` 中作为领域知识
- **Skills（独立技能）**：`weather-svg-creator` 由命令通过 Skill 工具直接调用

---

## 组件总结

| 组件 | 角色 | 示例 |
|------|------|------|
| **Command** | 入口点，用户交互 | `/weather-orchestrator` |
| **Agent** | 使用预加载技能获取数据 | `weather-agent` + `weather-fetcher` |
| **Skill** | 独立创建输出 | `weather-svg-creator` |

## 执行流程

1. **用户调用**：用户运行 `/weather-orchestrator` 命令
2. **用户提示**：命令询问用户偏好的温度单位（摄氏/华氏）
3. **Agent 调用**：命令通过 Agent 工具调用 `weather-agent`
4. **Skill 执行**（在 Agent 上下文中）：
   - Agent 按照 `weather-fetcher` 技能指令从 Open-Meteo 获取温度
   - Agent 将温度值和单位返回给命令
5. **SVG 创建**：命令通过 Skill 工具调用 `weather-svg-creator`
   - 创建 SVG 天气卡片
   - 写入摘要到 output.md
6. **结果展示**：向用户显示温度、SVG 卡片位置等信息

## 两种 Skill 模式对比

### Agent Skill（预加载模式）

```yaml
# 在 Agent 定义中 (.claude/agents/weather-agent.md)
---
name: weather-agent
skills:
  - weather-fetcher    # 启动时预加载到 Agent 上下文
---
```

- 技能被预加载：完整内容在启动时注入到 Agent 上下文中
- Agent 使用技能知识：Agent 按照预加载技能的指令执行
- 无动态调用：技能是参考材料，不单独调用

### Skill（直接调用模式）

```yaml
# 在 Skill 定义中 (.claude/skills/weather-svg-creator/SKILL.md)
---
name: weather-svg-creator
description: 创建 SVG 天气卡片...
---
```

- 通过 Skill 工具调用：命令调用 `Skill(skill: "weather-svg-creator")`
- 独立执行：在命令的上下文中运行，不在 Agent 内部
- 从上下文接收数据：使用对话中已有的温度数据

---

## 关键设计原则

1. **两种 Skill 模式**：同时展示预加载技能和直接调用技能
2. **Command 作为编排器**：命令处理用户交互并协调工作流
3. **Agent 负责数据获取**：Agent 使用预加载技能获取数据
4. **Skill 负责输出**：SVG 创建器独立运行，从命令上下文接收数据
5. **清晰分离**：获取（Agent）→ 渲染（Skill）— 每个组件单一职责
