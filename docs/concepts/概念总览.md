# 概念总览

> 来源：[shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice)

Claude Code 提供了一套完整的扩展体系，以下是所有核心概念模块的系统梳理。

---

## 核心功能模块

| 功能 | 位置 | 描述 |
|------|------|------|
| **Commands（命令）** | `.claude/commands/<name>.md` | 注入到现有上下文中的知识 — 用户调用的简单提示模板，用于工作流编排 |
| **Subagents（子代理）** | `.claude/agents/<name>.md` | 在全新隔离上下文中的自主执行者 — 拥有自定义工具、权限、模型、记忆和持久身份 |
| **Skills（技能）** | `.claude/skills/<name>/SKILL.md` | 注入到现有上下文中的知识 — 可配置、可预加载、可自动发现，支持上下文分叉和渐进式披露 |
| **Workflows（工作流）** | `.claude/commands/weather-orchestrator.md` | Command → Agent → Skill 编排模式 |
| **Hooks（钩子）** | `.claude/hooks/` | 在代理循环之外、特定事件上运行的确定性脚本 |
| **MCP Servers** | `.claude/settings.json`, `.mcp.json` | Model Context Protocol — 连接外部工具、数据库和 API |
| **Plugins（插件）** | 可分发包 | Skills、Subagents、Hooks 和 MCP Servers 的捆绑包，支持市场分发 |
| **Settings（设置）** | `.claude/settings.json` | 分层配置系统 — 权限、模型配置、输出风格、沙箱、快捷键、快速模式等 |
| **Status Line（状态栏）** | `.claude/settings.json` | 可自定义状态栏，显示上下文用量、模型、成本和会话信息 |
| **Memory（记忆）** | `CLAUDE.md`, `.claude/rules/`, `~/.claude/rules/` | 通过 CLAUDE.md 文件和 `@path` 导入实现的持久上下文 |
| **Checkpointing（检查点）** | 自动（基于 git） | 自动跟踪文件编辑，支持 `Esc Esc` 或 `/rewind` 回退和定向摘要 |
| **CLI Startup Flags（CLI 启动标志）** | `claude [flags]` | 命令行标志、子命令和环境变量 |

## 三者关系：Command vs Agent vs Skill

| 维度 | Command | Agent (Subagent) | Skill |
|------|---------|-------------------|-------|
| **上下文** | 注入到现有上下文 | 全新隔离上下文 | 注入到现有上下文（或 fork） |
| **调用方式** | 用户通过 `/command` 调用 | 通过 Agent 工具调用 | 通过 Skill 工具调用或自动发现 |
| **用途** | 工作流编排入口 | 自主执行复杂任务 | 知识注入/渐进式披露 |
| **隔离性** | 低 | 高（独立上下文） | 可选（context: fork） |
| **持久性** | 无 | 支持记忆（memory） | 无 |
| **适用场景** | 简单工作流模板 | 需要独立工具/权限的复杂任务 | 领域知识、操作指南 |

### 选择建议

- **小任务**：直接用 vanilla Claude Code，无需任何工作流
- **重复工作流**：使用 Commands 做编排入口
- **需要额外上下文的功能**：使用 Subagents + Skills（预加载）
- **通用知识/操作指南**：使用独立 Skills

> Boris Cherny 建议：工作流用 Commands 编排，Subagents 应该是功能特定的（而非通用的 "QA Agent"、"Backend Agent"），Skills 用于渐进式知识注入。

## 热门新特性

| 特性 | 位置 | 描述 |
|------|------|------|
| **/btw** | `/btw` | Claude 工作时的侧链对话 |
| **Code Review** | GitHub App | 多代理 PR 分析，捕获 bug、安全漏洞和回归 |
| **Scheduled Tasks** | `/loop` | 按计划运行提示（最长 3 天），轮询部署和构建 |
| **Voice Mode** | `/voice` | 语音输入提示 |
| **Simplify & Batch** | `/simplify`, `/batch` | 内置技能 — 简化重构、批量跨文件操作 |
| **Agent Teams** | 内置（环境变量） | 多代理并行在同一代码库上工作 |
| **Remote Control** | `/remote-control`, `/rc` | 从任何设备继续本地会话 |
| **Git Worktrees** | 内置 | 隔离的 git 分支用于并行开发 |
| **Ralph Wiggum Loop** | 插件 | 长时间自主开发循环 — 迭代直到完成 |
