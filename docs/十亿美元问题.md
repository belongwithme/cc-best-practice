# 十亿美元问题

> 来源：[shanraisshan/claude-code-best-practice](https://github.com/shanraisshan/claude-code-best-practice) — BILLION DOLLAR QUESTIONS 部分

Claude Code 生态中尚未解决的关键问题。

---

## 未解决的问题

### 1. 调试一致性
**Claude 能否始终如一地调试复杂问题？**

- 当前 Claude 在简单 bug 上表现出色
- 但在多层依赖、竞态条件等复杂场景中一致性不够
- CLAUDE.md 中的调试指令不保证 100% 执行

### 2. 长上下文质量
**上下文窗口增长后，输出质量是否能保持？**

- 当前经验：上下文越长，输出质量越容易下降
- `/compact` 是缓解方案但不是根本解决
- 这是所有 LLM 面临的基础挑战

### 3. 多代理协调
**Agent Teams 能否在真正复杂的项目中可靠工作？**

- 当前 Agent Teams 在简单并行任务上有效
- 代理间的依赖和冲突解决仍需人工介入
- Git merge 冲突是主要瓶颈

### 4. 安全性
**AI 生成的代码是否引入了新的安全风险？**

- Claude Code Review 可以检测已知模式
- 但新型攻击向量（如 prompt injection via code）仍是挑战
- 团队需要建立 AI 代码审查的安全标准

### 5. 可重复性
**相同的提示能否产生一致的结果？**

- LLM 的本质决定了输出的随机性
- 通过 CLAUDE.md、Skills、Commands 可以提高一致性
- 但 100% 可重复性目前不可达
